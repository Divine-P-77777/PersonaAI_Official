## Backend Analysis

### Current State (Phase 1 - Completed)
- **Architecture**: Decoupled FastAPI (Backend) and Next.js (Frontend) linked via high-speed REST and SSE.
- **Core Features**:
    - **Ingestion Pipeline**: Automated processing of diverse data sources with async workers.
    - **Persona Creation**: Deep customization of personality, expertise, and custom knowledge.
    - **Chat Interface**: Ultra-fast, RAG-based chat using **SSE** for token streaming.
- **Authentication**: JWT-based secure session management via Supabase Auth.
- **RAG Pipeline**:
    - **OCR**: `Tesseract OCR` (Windows Native Path) & `PyMuPDF` for high-fidelity scanning.
    - **Embeddings**: `Nomic` (v1.5 - 768d) for state-of-the-art semantic representation.
    - **Vector Storage**: Supabase (PostgreSQL with `pgvector`) for sub-millisecond similarity search.
    - **Conversation Context**: Persistent **Top-5 Message Rolling Window** for stable dialogue memory.
- **Storage Strategy**:
    - **Cloudinary**: Dedicated high-bandwidth storage for **Ingested Documents** (PDFs/Source files).
    - **Supabase Bucket**: Isolated bucket storage for **Persona Avatars** and Profile Images.
- **Seamless Production**: **Dockerized Containerization** for both Backend and Ingestion workers, ensuring environment parity across all stages.

### Future Enhancement & Robustness (Phase 2 - Ongoing)
- **Live Talk Mode**: Real-time voice interaction using **WebSockets**.
    - **STT**: **Deepgram** (Ultra-low latency transcription).
    - **TTS**: **ElevenLabs** (Premium English) & **Sarvam AI** (Human-like Hindi/Indic).
- **Two-Stage Reranking**: Implementing the **Cohere Reranking Model** (Retrieve 15 candidates -> Filter to top 5) for maximum context accuracy.
- **Advanced Workers**: Distributed task processing using Celery & RabbitMQ for heavy document ingestion.

## Proposed PPT Structure (10 Slides)

### 1. Title Slide
- Project Name: **PersonaBot: AI-Powered Knowledge Base & Custom Personas**
- Subtitle: A Multi-tenant RAG-based Chatbot Platform

### 2. Introduction & Problem Statement
- **Problem**: Static documents are hard to navigate; generic AI lacks human personality.
- **Solution**: "Personify" knowledge—letting users chat with the AI version of subject matter experts.

### 3. System Architecture: The Backend Hub
- Detailed view of FastAPI connecting to Groq (Llama 3.3 70B), Supabase, and Cloudinary.
- **Chat vs Live Mode** communication protocols (SSE vs WebSockets).

### 4. The RAG Engine: Ingestion to Retrieval
- OCR (Tesseract) -> Chunking -> Nomic Embeddings -> pgvector Search.

### 5. Perfected Business Model & Monetization
- **Revenue Model**:
    - **B2C Subscription**: Monthly "Pro" plans for student access to premium mentors.
    - **B2B Licensing**: Corporate/University licenses for internal expert personas.
- **Marketplace Creator Model**:
    - **Revenue Share**: Persona creators (Alumni/Profs) earn 70% of subscription revenue generated by their bots.
    - **Free vs. Paid Tiers**: Creators decide if their knowledge is open-access or paywalled.
- **Integration**: Automated payouts and transaction handling via **Cashfree Gateway**.

### 6. Technical Stack: High-Speed AI
- **Inference**: Groq (llama-3.3-70b-versatile).
- **Voice**: Deepgram (STT) & Sarvam AI (TTS).
- **Memory**: Persistent Rolling Window (Top-5 Messages).

### 7. Security: Row Level Isolation
- Multi-tenant data safety using Supabase RLS and `bot_id` schema filtering.

### 8. Deployment Strategy
- **Frontend**: Vercel.
- **Backend**: AWS EC2 (Dual-Instance for HA).
    - **Containerization**: **Docker Image** based deployment for automated scaling and zero-dependency overhead on host machines.

### 9. Future Roadmap: Phase II & Beyond
- Global multi-lingual voice support; Distributed worker nodes; Cohere Reranking model.

### 10. Conclusion & Q&A
- Summary of the multi-persona ecosystem.
